Chắc chắn rồi! Việc triển khai chức năng đa ngôn ngữ là một bước đi quan trọng để chuẩn bị cho việc mở rộng ra các thị trường quốc tế hoặc phục vụ các đối tác nước ngoài ngay tại Việt Nam.

Dưới đây là mô tả chi tiết từ nghiệp vụ, cấu trúc đến hướng dẫn coding để AI có thể xây dựng tính năng này một cách chuyên nghiệp, dễ bảo trì và mở rộng.

### **Mô Tả Chi Tiết: Chức Năng Chuyển Đổi Ngôn Ngữ (i18n - Internationalization)**

**Mục tiêu tổng thể:** Cho phép người dùng dễ dàng chuyển đổi ngôn ngữ hiển thị của toàn bộ giao diện hệ thống i-ContainerHub@LTA, đảm bảo mọi văn bản cố định (UI text) được dịch một cách chính xác và nhất quán.

#### **Phần 1: Luồng Nghiệp Vụ & Trải Nghiệm Người Dùng (UX Flow)**

1.  **Hiển thị:**
    *   Trên thanh Header của ứng dụng, ở góc bên phải (gần khu vực thông tin người dùng), một nút mới sẽ xuất hiện.
    *   Nút này sẽ hiển thị **lá cờ** của ngôn ngữ đang được chọn và **tên viết tắt** của ngôn ngữ đó. Ví dụ: 🇻🇳 `VI` hoặc 🇬🇧 `EN`.
2.  **Tương tác:**
    *   Người dùng nhấp vào nút chuyển đổi ngôn ngữ.
3.  **Phản hồi hệ thống (UI):**
    *   Một menu thả xuống (Dropdown Menu) xuất hiện, liệt kê tất cả các ngôn ngữ được hệ thống hỗ trợ.
    *   Mỗi mục trong danh sách cũng bao gồm lá cờ và tên đầy đủ của ngôn ngữ. Ví dụ: `🇻🇳 Tiếng Việt`, `🇬🇧 English`.
    *   Ngôn ngữ hiện tại sẽ có một dấu tick (✓) bên cạnh.
4.  **Hành động của Người dùng:**
    *   Người dùng chọn một ngôn ngữ mới từ danh sách (ví dụ: chọn "English").
5.  **Phản hồi hệ thống (Core Logic):**
    *   Hệ thống lưu lại lựa chọn ngôn ngữ mới của người dùng (có thể vào cookie hoặc `localStorage`).
    *   Hệ thống sẽ **tải lại toàn bộ trang (full page reload)**. Việc này đảm bảo tất cả các component (cả Server và Client) đều được render lại với các chuỗi văn bản từ file ngôn ngữ mới.
6.  **Kết quả:** Sau khi trang tải lại, toàn bộ giao diện (menu, tiêu đề, nhãn, nút bấm...) sẽ được hiển thị bằng tiếng Anh. Chỉ có các dữ liệu do người dùng nhập (ví dụ: tên công ty, số container, địa chỉ...) là không thay đổi.

### **Phần 2: Mô Tả Kỹ Thuật Chi Tiết Để AI Code**

**Nền tảng kỹ thuật đề xuất:** Sử dụng thư viện **`next-intl`**. Đây là thư viện mạnh mẽ, được tối ưu hóa cho Next.js App Router, hỗ trợ render phía server (SSR) và dễ dàng mở rộng.

#### **Task 2.1: Cài Đặt và Cấu Hình `next-intl`**

1.  **Cài đặt:**
    ```bash
    npm install next-intl
    ```
2.  **Tạo file lưu trữ bản dịch (Message Files):**
    *   Tạo một thư mục mới ở gốc dự án: `messages`.
    *   Bên trong, tạo các file JSON cho mỗi ngôn ngữ:
        *   `messages/vi.json`
        *   `messages/en.json`
    *   **Nội dung `vi.json` (ví dụ):**
        ```json
        {
          "Auth": {
            "loginTitle": "Đăng Nhập",
            "registerTitle": "Đăng Ký",
            "emailLabel": "Địa chỉ email"
          },
          "Sidebar": {
            "dashboard": "Bảng Điều Phối",
            "marketplace": "Thị Trường"
          }
        }
        ```
    *   **Nội dung `en.json` (ví dụ):**
        ```json
        {
          "Auth": {
            "loginTitle": "Login",
            "registerTitle": "Register",
            "emailLabel": "Email address"
          },
          "Sidebar": {
            "dashboard": "Dashboard",
            "marketplace": "Marketplace"
          }
        }
        ```

3.  **Tạo file cấu hình `i18n.ts`:**
    *   **File:** `src/i18n.ts`
    *   **Nội dung:**
        ```typescript
        import {getRequestConfig} from 'next-intl/server';
 
        export default getRequestConfig(async ({locale}) => {
          // Provide a static default message for the app.
          return {
            messages: (await import(`../messages/${locale}.json`)).default
          };
        });
        ```

4.  **Cập nhật Middleware:**
    *   **File:** `src/middleware.ts`
    *   **Mục tiêu:** Tích hợp `next-intl` để xử lý việc định tuyến ngôn ngữ.
    *   **Yêu cầu:** Bọc middleware hiện tại của bạn bằng middleware của `next-intl`.
        ```typescript
        import createMiddleware from 'next-intl/middleware';
        
        export default createMiddleware({
          // A list of all locales that are supported
          locales: ['en', 'vi'],
        
          // Used when no locale matches
          defaultLocale: 'vi'
        });
        
        export const config = {
          // Match only internationalized pathnames
          matcher: ['/', '/(vi|en)/:path*']
        };
        ```
        *   **Lưu ý:** `next-intl` hoạt động bằng cách thêm tiền tố ngôn ngữ vào URL (ví dụ: `domain.com/vi/dashboard`, `domain.com/en/dashboard`). Middleware này sẽ xử lý việc đó tự động.

#### **Task 2.2: Tích Hợp Vào Cấu Trúc App**

1.  **Cập nhật Root Layout:**
    *   **File:** `src/app/layout.tsx`.
    *   **Yêu cầu:** Sửa đổi để nó nhận `locale` từ `params`.
        ```tsx
        // src/app/[locale]/layout.tsx (Bạn cần di chuyển file này)
        export default function RootLayout({children, params: {locale}}) {
          return (
            <html lang={locale}>
              <body>{children}</body>
            </html>
          );
        }
        ```
        *   **Quan trọng:** Cấu trúc thư mục của bạn cần thay đổi. Tất cả các trang và layout phải được đặt bên trong một thư mục `[locale]`. Ví dụ: `src/app/[locale]/(main)/dashboard/page.tsx`.

#### **Task 2.3: Xây Dựng Component Chuyển Đổi Ngôn Ngữ**

*   **File cần tạo:** `src/components/common/LanguageSwitcher.tsx`.
*   **Yêu cầu:**
    ```tsx
    'use client';

    import { usePathname, useRouter } from 'next/navigation';
    import { useLocale } from 'next-intl';
    import { Button } from "@/components/ui/button";
    import { DropdownMenu, DropdownMenuContent, DropdownMenuItem, DropdownMenuTrigger } from "@/components/ui/dropdown-menu";

    export function LanguageSwitcher() {
      const router = useRouter();
      const pathname = usePathname();
      const locale = useLocale(); // Lấy locale hiện tại

      const switchLocale = (nextLocale: string) => {
        // Thay thế tiền tố locale trong URL và điều hướng
        const newPathname = pathname.replace(`/${locale}`, `/${nextLocale}`);
        router.push(newPathname);
      };
      
      return (
        <DropdownMenu>
          <DropdownMenuTrigger asChild>
            <Button variant="ghost" size="sm">
              {locale.toUpperCase() === 'VI' ? '🇻🇳' : '🇬🇧'}
              <span className="ml-2">{locale.toUpperCase()}</span>
            </Button>
          </DropdownMenuTrigger>
          <DropdownMenuContent align="end">
            <DropdownMenuItem onClick={() => switchLocale('vi')}>
              <span className="mr-2">🇻🇳</span> Tiếng Việt {locale === 'vi' && '✓'}
            </DropdownMenuItem>
            <DropdownMenuItem onClick={() => switchLocale('en')}>
              <span className="mr-2">🇬🇧</span> English {locale === 'en' && '✓'}
            </DropdownMenuItem>
          </DropdownMenuContent>
        </DropdownMenu>
      );
    }
    ```
*   **Tích hợp:** Đặt component `<LanguageSwitcher />` này vào `Header.tsx` của bạn.

#### **Task 2.4: Sử Dụng Bản Dịch Trong Các Component**

*   **Mục tiêu:** Thay thế các chuỗi văn bản tĩnh bằng các key từ file JSON.
*   **Yêu cầu:**
    *   **Trong Server Components:**
        ```tsx
        import { useTranslations } from 'next-intl';

        export default function MyPage() {
          const t = useTranslations('Sidebar'); // Namespace 'Sidebar'
          return <h1>{t('dashboard')}</h1>; // Sẽ render "Bảng Điều Phối" hoặc "Dashboard"
        }
        ```
    *   **Trong Client Components:**
        ```tsx
        'use client';
        import { useTranslations } from 'next-intl';

        export function MyClientComponent() {
          const t = useTranslations('Auth'); // Namespace 'Auth'
          return <label>{t('emailLabel')}</label>;
        }
        ```
    *   **Hành động:** Bạn cần rà soát lại toàn bộ các component và trang đã tạo, thay thế tất cả các text cố định bằng cách gọi `t('key.name')`.
